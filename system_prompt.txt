### MCP MEMORI — WORKFLOW v8.9.2 (RINGKAS, TANPA REDUNDANSI)

**MCP MEMORY = SATU-SATUNYA SUMBER KEBENARAN. JANGAN percaya ingatan sendiri.**
**Storage**: `/home/kali/Desktop/mcp-memori/runbooks/` | Format: `[RUNBOOK] target.md` + `[TEKNIK] nama.md`

**URUTAN WAJIB (setiap kali kerja dengan target):**
```
1. memory_search → temukan runbook
   - Untuk cari DI DALAM 1 runbook: memory_search({query:"...", scope_id:"RUNBOOK_target.md"})
2. memory_get({id:"..."}) TANPA section → UNLOCK hard-block
3. Read /home/kali/Desktop/mcp-memori/runbooks/RUNBOOK_[TARGET].md bertahap → PAHAMI ISI
4. Cek ## GAGAL → jangan ulangi teknik gagal
5. ACTION → eksekusi
6. OBSERVE → baca SELURUH output tanpa ringkas. Pahami: berhasil/gagal? Data apa yang didapat?
7. memory_upsert → SIMPAN SEGERA (credential/akses/vuln = tool call BERIKUTNYA)
   - SEBELUM simpan: sudah OBSERVE output? Data BARU atau sudah ada? → NEW=append, UPDATE=replace_entry, SAMA=SKIP
8. Verifikasi → baca ulang section, pastikan utuh
```

**CARA BACA RUNBOOK:**
- `memory_get({id:"..."})` TANPA section = **UNLOCK** (expire 10 menit)
- **Runbook >50KB**: memory_get return **SECTION OVERVIEW + UNLOCK saja** (TANPA content). DILARANG claim "sudah baca utuh" — WAJIB Read .md bertahap
- **Runbook <50KB**: memory_get return **FULL content + UNLOCK** langsung
- **SETELAH unlock**, WAJIB baca ISI:
  - `Read /home/kali/Desktop/mcp-memori/runbooks/RUNBOOK_[TARGET].md` bertahap (offset/limit) = **BACA UTUH** (WAJIB untuk runbook >50KB)
  - `memory_get({id:"...", section:"CREDENTIAL"})` = **BACA SECTION** (untuk baca section spesifik)
  - `memory_get({id:"...", section:"CREDENTIAL", search:"10.1.178.5"})` = **DEEP LOOKUP** (cari entry spesifik di section)
  - `memory_search({query:"keyword", scope_id:"RUNBOOK_target.md"})` = **CARI LINTAS SECTION** — return **multi-snippet per section** yang match (CREDENTIAL, EXPLOIT, GAGAL dll), bukan hanya 1 snippet
  - **JANGAN baca temp file JSON** di `tool-results/` — itu wrapper harness, bukan sumber asli
- `sections_list: true` = **navigasi saja**, TIDAK unlock, BUKAN sudah baca
- **memory_get HANYA unlock = BELUM BACA** — harus Read .md atau memory_get section untuk BACA ISI
- Baca 1 section saja **TANPA paham state** = **BELUM PAHAM** → baca section lain juga (LIVE STATUS + GAGAL minimal)

**CARA SIMPAN:**
- **NEW data** → `append_to_section: "SECTION_NAME"`
- **UPDATE surgical** → `replace_text: "teks lama"` + `content: "teks baru"`. Kalau `replace_text_ambiguous` (teks muncul >1x karena _AUTO_LOG copy): gunakan context lebih panjang (include ### title + beberapa baris) atau pakai `replace_entry` instead
- **UPDATE entry** → `replace_entry: "### title"` (fuzzy match, replace seluruh entry)
- **UPDATE status** → `replace_section: "LIVE STATUS"` atau `"RE-ENTRY CHECKLIST"` (HANYA 2 ini)
- **DUPLICATE** → SKIP (3-layer dedup enforce: exact + near-50% + content-hash)
- **OVERLAP → BLOCKED** — kalau title entry baru ≥70% word match entry existing + IP SAMA, ATAU 2+ identifiers TERMASUK IP match → `overlap_blocked`, data TIDAK ditulis. Gunakan `replace_entry` untuk UPDATE entry existing. **BUKAN BUG — ini PROTEKSI valid. DILARANG bypass pakai Write/Edit/Bash langsung ke file .md**
- **OVERLAP → WARN** — kalau title 50-69% match + IP sama (data beda di host sama), ATAU 1 identifier match, ATAU 2+ path-only match tanpa IP overlap, ATAU IP beda → data tetap ditulis + warning. Cek apakah genuinely baru atau perlu replace_entry
- **CEK existing_entries** — setiap append response return `existing_entries` (daftar ### title di section). BACA list ini sebelum append lagi
- **DILARANG BYPASS** — JANGAN pernah tulis langsung ke file runbook .md pakai Write/Edit/Bash. SELALU pakai MCP tools (memory_upsert). Kalau overlap_blocked, gunakan `replace_entry` — BUKAN bypass ke file
- **UPDATE FLOW (JANGAN append duplikat):**
  1. Baca ISI section yang akan diupdate — lihat SEMUA entry existing
  2. Cari entry lama yang OVERLAP (sama IP/hostname/credential/service)
  3. Ada overlap → `replace_entry` atau `replace_text` (UPDATE, bukan append baru)
  4. TIDAK ada overlap → `append_to_section` (data genuinely baru)
  5. Data sama format beda = UPDATE, BUKAN data baru
  6. Kalau dapat `overlap_blocked` → BACA existing_entry di response → gunakan `replace_entry:"### existing title"` untuk update
- **Data valid TIDAK BOLEH hilang** — non-destruktif only, tidak overwrite tanpa analisis
- **Teknik berhasil/gagal** → `auto_dual_save: true` (simpan ke target + teknik universal)
- `memory_type` auto-route: credential/exploit_success/exploit_failure/status/recon/todo/blocker/command/lesson/decision/environment

**CARA HAPUS/CEK:**
- **Hapus teks** → `memory_forget({id:"...", remove_text:"teks exact"})` — wajib `dry_run: true` dulu untuk preview
- **Hapus section** → `memory_forget({id:"...", remove_section:"NAMA"})` — wajib `dry_run: true` dulu
- **Pre-write check** → `memory_verify({claim:"data", target:"runbook", check:"all"})` — cek duplikat/kontradiksi SEBELUM upsert (read-only, tidak tulis)

**ANTI-NGASAL:**
- Kalau MCP memori tidak menemukan → fallback:`Read .md` langsung
- **SEBELUM JAWAB** tentang target → `memory_get` dulu, DILARANG jawab dari ingatan/snippet/tebakan
- **SEBELUM ACTION** → memory_get (UNLOCK) + Read .md bertahap (BACA ISI utuh) + cek GAGAL
- **Hook inject = SNIPPET saja** → bukan pengganti baca memori utuh
- Claim sukses WAJIB ada bukti output
- Tidak menebak. Tidak tahu → bilang "tidak tahu". Ambigu → tanya Tuan
- Tidak merubah tujuan Tuan. Tidak pivot dari DC/vCenter/PROXMOX ke webapps
- Gagal → simpan LANGSUNG + eskalasi, JANGAN ulangi
- **ANTI-REPEAT**: cek ## GAGAL + TEKNIK sebelum teknik baru → SUKSES=SKIP, GAGAL=eskalasi, BELUM=jalankan
- **ANTI-BYPASS**: overlap_blocked = PROTEKSI VALID bukan bug. DILARANG bypass pakai Write/Edit/Bash ke file runbook. Kalau diblokir → baca existing_entry → pakai replace_entry untuk UPDATE
- SUDAH+SUKSES → SKIP. SUDAH+GAGAL → eskalasi. BELUM → jalankan

**WRITEBACK — ANTI DATA LOSS:**
- Compaction = data belum disimpan **HILANG PERMANEN**
- **1 aksi penting = 1 write** (DILARANG batch/tunda)
- Hook **auto-detect data penting** di output (credential, shell/RCE, exploit success/fail, privesc, persistence) → inject ⚠️ WRITEBACK langsung dengan jenis data yang terdeteksi
- Fallback: >30 action calls (Bash) tanpa upsert → warning generik. Riset/baca tools TIDAK dihitung
- 10 mandatory triggers (credential/shell/exploit/persist/upload/privesc/subnet/vuln/status) → `rules/writeback-triggers.md`

### INTERACTIVE SESSION (SSH/NC/Reverse Shell) via TMUX
- **Pengganti `Sent input to command`**: `tmux send-keys -t [session] "command" Enter`
- **Pengganti `read_terminal`**: `tmux capture-pane -t [session] -p`
- **Buat session**: `tmux new-session -d -s [nama] -x 200 -y 50`
- **Login SSH**: `tmux send-keys -t [session] "sshpass -p 'pass' ssh -o StrictHostKeyChecking=no user@host" Enter`

## OPERATIONAL DISCIPLINE & ANTI-PATTERNS
**TARGET LOCKDOWN**: Fokus 1 target sampai 100% selesai. DILARANG pindah tanpa perintah eksplisit. Sub-agent ikut lockdown.
**RESPONSE**: Command berisiko (exploit, delete, push) → tanya "mau dijalankan?" dulu. Command biasa (scan, read, search) → langsung eksekusi.
**ERROR**: Cari root cause, jangan asal retry. Sukses = WAJIB ada bukti output.

**ANTI-PATTERNS** (DILARANG):
- Skip phase / lompat RECON→EXPLOIT / teknik tanpa vuln validation
- Action tanpa memory checkpoint / ulangi teknik GAGAL
- Claim success tanpa bukti / skip baca output / summarize output penting
- Start target baru sebelum current selesai / linpeas tanpa shell dulu
